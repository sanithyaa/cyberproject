About SecureVote

A cybersecurity-focused online voting platform

Security Architecture
  • bcrypt password hashing (cost factor 12)
  • Flask-WTF CSRF protection on all forms
  • SQLAlchemy ORM – parameterised queries only
  • Jinja2 auto-escaping prevents XSS
  • Role-based access control (Admin / Voter)
  • Rate limiting on login (brute-force prevention)
  • TOTP-based OTP two-factor authentication
  • Secure HTTP headers (CSP, X-Frame-Options, etc.)
  • Session timeout after 30 minutes
  • Immutable audit trail for all actions
Tech Stack
  • Python Flask (backend)
  • SQLite / MySQL (database)
  • Bootstrap 5 (UI)
  • Chart.js (analytics)
  • Flask-WTF (CSRF)
  • Flask-Bcrypt (hashing)
  • Flask-Login (sessions)
  • Flask-Limiter (rate limiting)
Attack Prevention
SQL Injection
' OR '1'='1

Blocked by SQLAlchemy ORM parameterised queries.

XSS Attack
<script>alert(1)</script>

Blocked by Jinja2 auto-escaping and input sanitisation.

CSRF Attack
Forged form submission

Blocked by Flask-WTF CSRF tokens on every form.
